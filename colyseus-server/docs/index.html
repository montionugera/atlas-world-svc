üì® Message Types

üì§ Client ‚Üí Server Messages

SEND player_input

Send player movement input (velocity)

{ "type": "player_input", "data": { "vx": 0.5, // Velocity X (-1 to 1) "vy": -0.3 // Velocity Y (-1 to 1) } }

SEND player_position

Send direct player position update

{ "type": "player_position", "data": { "x": 150.5, // X coordinate "y": 200.3 // Y coordinate } }

üì• Server ‚Üí Client Messages

RECEIVE welcome

Message sent when player joins the room

{ "type": "welcome", "message": "Welcome to map-01-sector-a!", "playerId": "abc123def456", "mapId": "map-01-sector-a" }

RECEIVE state_change

Real-time game state updates

{ "type": "state_change", "players": { "player1": { "id": "player1", "sessionId": "abc123", "x": 150.5, "y": 200.3, "vx": 0.5, "vy": -0.3, "name": "Player1" } }, "mobs": [ { "id": "mob-1", "x": 300.0, "y": 150.0, "vx": 1.0, "vy": 0.5 } ], "tick": 1234, "mapId": "map-01-sector-a" }

üìã Data Schemas

Player Object

string id required - Player ID
string sessionId required - WebSocket session ID
number x required - X coordinate
number y required - Y coordinate
number vx required - Velocity on X axis
number vy required - Velocity on Y axis
string name required - Player display name

Mob Object

string id required - Mob ID
number x required - X coordinate
number y required - Y coordinate
number vx required - Velocity on X axis
number vy required - Velocity on Y axis
